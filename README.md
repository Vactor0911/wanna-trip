## 📖 프로젝트 개요

**여행갈래(Wanna Trip)** 는 유튜브, 블로그, 검색 등 여러 출처에 분산된 여행 정보를 통합 관리하고, 서로 다른 형식의 정보를 일관된 포맷으로 정리할 수 있는 **AI 기반 여행 계획 플랫폼**입니다.

### 핵심 가치

| 가치 | 설명 |
|------|------|
| **🗂️ 정보 통합 관리** | 분산된 여행 정보를 하나의 플랫폼에서 체계적으로 관리 |
| **📋 템플릿 공유 시스템** | 커뮤니티에서 검증된 여행 계획을 즉시 활용 가능 |
| **🤖 AI 기반 자동화** | 여행 정보 입력 시 AI가 최적화된 초기 일정을 자동 생성 |
| **👥 실시간 협업** | Socket.IO 기반 다중 사용자 동시 편집 지원 |

## 🚀 주요 기능

### 3.1 사용자 인증 및 계정 관리

#### 회원가입
| 기능 | 설명 |
|------|------|
| **이메일 인증** | 6자리 인증 코드 발송, 5분 유효 시간 타이머 표시 |
| **비밀번호 유효성 검사** | 8~20자, 영문+숫자+특수문자 조합 필수 |
| **비밀번호 확인** | 실시간 일치 여부 확인 |
| **이용약관 동의** | 서비스 이용약관, 개인정보처리방침 동의 체크 |
| **중복 검사** | 이메일 중복 가입 방지 |

#### 로그인 / 로그아웃
| 기능 | 설명 |
|------|------|
| **일반 로그인** | 이메일 + 비밀번호 기반 JWT 토큰 인증 |
| **카카오 로그인** | Kakao OAuth 2.0 간편 로그인 |
| **구글 로그인** | Google OAuth 2.0 간편 로그인 |
| **자동 토큰 갱신** | Access Token 만료 시 Refresh Token으로 자동 갱신 (`TokenRefresher` 컴포넌트) |
| **로그아웃** | 서버 측 Refresh Token 무효화 + 클라이언트 쿠키 삭제 |

#### 계정 연동
| 기능 | 설명 |
|------|------|
| **소셜 → 일반 연동** | 소셜 계정에 비밀번호 설정하여 일반 로그인 활성화 |
| **일반 → 소셜 연동** | 기존 계정에 카카오/구글 계정 연결 |
| **연동 충돌 처리** | 동일 이메일 존재 시 계정 연동 다이얼로그 표시 |

#### 비밀번호 찾기 / 변경
| 기능 | 설명 |
|------|------|
| **비밀번호 찾기** | 이메일 인증 후 새 비밀번호 설정 |
| **비밀번호 변경** | 현재 비밀번호 확인 후 새 비밀번호로 변경 |
| **인증 코드 재발송** | 5분 타이머 만료 시 재발송 가능 |

#### 프로필 관리
| 기능 | 설명 |
|------|------|
| **프로필 이미지 업로드** | 이미지 파일 업로드 (자동 압축 처리) |
| **프로필 이미지 삭제** | 기본 이미지로 초기화 |
| **닉네임 변경** | 2~10자 닉네임 수정 |
| **계정 삭제** | 비밀번호 확인 후 계정 영구 삭제 |

#### 인기 템플릿 & 공유 템플릿
| 기능 | 설명 |
|------|------|

| **공유 템플릿** | 나에게 공유된 템플릿 목록 조회 (`useSharedTemplates`) |

### 3.5 네이버 지도 통합

#### 지도 컴포넌트 (`NaverMap`)
| 기능 | 설명 |
|------|------|
| **인터랙티브 모드** | 줌, 패닝, 마커 클릭 등 사용자 상호작용 지원 |
| **정적 모드** | 카드 미리보기용 고정 지도 표시 |
| **마커 표시** | 선택된 장소를 지도에 마커로 표시 |
| **폴리라인** | 여러 장소 간 연결선 표시 (경로 시각화) |

| 기능 | 설명 |

| **장소 선택** | 검색 결과에서 장소 클릭하여 선택 |
| **위치 정보 반환** | 선택된 장소의 이름, 주소, 좌표, 카테고리, 썸네일 반환 |

#### 위치 정보 구조 (`LocationInterface`)
```typescript
{
  name: string;       // 장소명
  address: string;    // 주소
  lat: number;        // 위도
  lng: number;        // 경도
  category: string;   // 카테고리 (관광지, 맛집 등)
  thumbnail: string;  // 썸네일 이미지 URL
}
```

### 3.12 보안

#### 인증 보안
| 기능 | 설명 |
|------|------|
| **JWT 토큰** | Access Token (15분) + Refresh Token (7일) 이중 인증 |
| **HttpOnly 쿠키** | XSS 공격으로부터 토큰 보호 |
| **Secure 쿠키** | HTTPS에서만 쿠키 전송 |
| **SameSite 설정** | CSRF 공격 방지를 위한 쿠키 설정 |

#### 비밀번호 보안
| 기능 | 설명 |
|------|------|
| **bcrypt 해싱** | 비밀번호 단방향 해시 (salt rounds: 10) |
| **복잡도 검증** | 8~20자, 영문+숫자+특수문자 필수 |
| **재사용 방지** | 이전 비밀번호와 동일 여부 확인 |

#### 서버 보안
| 기능 | 설명 |
|------|------|
| **Helmet.js** | 보안 HTTP 헤더 자동 설정 |
| **CSRF 토큰** | 상태 변경 요청 시 CSRF 토큰 검증 |
| **Rate Limiting** | API 요청 횟수 제한 (express-rate-limit) |
| **CORS 설정** | 허용된 출처만 API 접근 가능 |
| **입력 검증** | express-validator로 모든 입력 데이터 검증 |

#### 데이터 보안
| 기능 | 설명 |
|------|------|
| **환경 변수** | 민감 정보 `.env` 파일로 분리 |
| **SQL Injection 방지** | Prepared Statement 사용 |
| **파일 업로드 제한** | 파일 크기, 확장자 제한 (Multer) |
